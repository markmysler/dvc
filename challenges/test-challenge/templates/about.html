About This Challenge

What is Cross-Site Scripting (XSS)?

Cross-Site Scripting (XSS) is a security vulnerability that occurs when a web application includes untrusted data in a web page without proper validation or escaping. This allows attackers to inject malicious scripts that execute in the victim's browser.

Types of XSS

  • Reflected XSS: Malicious script is reflected off a web server, such as in an error message or search result
  • Stored XSS: Malicious script is permanently stored on the target server
  • DOM-based XSS: Vulnerability exists in client-side code rather than server-side

This Challenge

This challenge focuses on Reflected XSS. The vulnerability exists in the search functionality where user input is reflected back to the page without proper sanitization.

Learning Objectives

  • Understand how XSS vulnerabilities occur
  • Learn to identify potential injection points
  • Practice crafting basic XSS payloads
  • Understand the impact and risks of XSS attacks

Prevention

XSS vulnerabilities can be prevented by:

  • Properly encoding/escaping user input
  • Using Content Security Policy (CSP) headers
  • Validating and sanitizing input on both client and server side
  • Using modern frameworks with built-in XSS protection

⚠️ Educational Use Only

This application contains intentional security vulnerabilities for educational purposes. Do not use these techniques on systems you do not own or without explicit permission.
